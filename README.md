# NEO STORE: Магазин реализации внутренней валюты

## Описание проекта

NEO STORE — корпоративный веб-сервис для управления внутренней валютой компании и мотивации сотрудников.

Сервис позволяет сотрудникам зарабатывать корпоративную валюту (NeoCoins) путём выполнения заданий и челленджей, а также приобретать товары в корпоративном магазине.

## Основные возможности

### Для сотрудников
- Регистрация и авторизация в системе
- Загрузка сертификатов (фото или PDF) за достижения
- Просмотр статуса сертификатов (ожидает/одобрено)
- Удаление необработанных сертификатов
- Просмотр баланса NeoCoins
- Покупка товаров в магазине
- Оформление заказов
- Просмотр истории заказов

### Для HR-менеджеров
- Просмотр ожидающих сертификатов сотрудников
- Одобрение сертификатов с начислением награды
- Запуск челленджей и награждение выбранных сотрудников
- Создание кошельков для новых пользователей
- Отметка заказов как доставленных

## Технологический стек

- **Backend:** Django 6.0
- **База данных:** SQLite
- **Frontend:** HTML, CSS, Bootstrap 5
- **Аутентификация:** Django Auth

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/w0nkest/neostore.git
cd neostore
```

### 2. Создание виртуального окружения
```bash
python -m venv .venv
```

### 3. Активация виртуального окружения
```bash
# Windows
.venv\Scripts\activate

# Linux/Mac
source .venv/bin/activate
```

### 4. Установка зависимостей
```bash
pip install django
```

### 5. Миграции базы данных
```bash
python neostore/manage.py migrate
```

### 6. Создание суперпользователя (HR-менеджера)
```bash
python neostore/manage.py createsuperuser
```

### 7. Запуск сервера
```bash
python neostore/manage.py runserver
```

Сервис будет доступен по адресу: http://127.0.0.1:8000/

## Модели данных

### User
- Имя пользователя
- Пароль
- Кошелёк
- Флаг суперпользователя (для HR)

### Certificate
- Пользователь
- Название
- За что выдан
- Файл сертификата (фото/PDF)
- Статус (одобрен/ожидает)
- Дата создания

### Thing
- Название
- Фото
- Цена
- Количество на складе

### Order
- Пользователь
- Дата создания
- Статус (доставлен/в обработке)
- Элементы заказа

### OrderItem
- Заказ
- Товар
- Количество
- Цена

### Wallet
- Пользователь
- Баланс

## Роли пользователей

### Сотрудник
- Загружает сертификаты за достижения
- Просматривает баланс NeoCoins
- Покупает товары в магазине
- Отслеживает заказы

### HR-менеджер (суперпользователь)
- Все возможности сотрудника
- Одобрение/отклонение сертификатов
- Начисление валюты за сертификаты
- Запуск челленджей и массовое награждение сотрудников
- Управление товарами в магазине
- Закрытие заказов после выдачи

## Интерфейс

Интерфейс локализован на русский язык и адаптирован для использования на корпоративном портале.

### Страницы
- **Главная** — приветствие и кнопка входа
- **Вход/Регистрация** — авторизация пользователей
- **Панель сотрудника** — баланс, сертификаты, заказы
- **Магазин** — каталог товаров
- **HR Панель** — управление сертификатами и заказами
- **Задачи** — награждение сотрудников

## Кейс для хакатона NEO.HACK()

Данный сервис разработан в рамках кейса "NEO STORE: Магазин реализации внутренней валюты" для хакатона NEO.HACK().

### Основные сценарии использования

1. **Сотрудник загружает сертификат** за достижение (конференция, сертификация и т.д.)
2. **HR проверяет и одобряет** сертификат, начисляя награду в NeoCoins
3. **HR запускает челлендж** и награждает выбранных сотрудников
4. **Сотрудник покупает товары** в магазине за заработанную валюту
5. **HR закрывает заказ** после физической выдачи товара
